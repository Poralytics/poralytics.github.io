SQL Injection Vulnerability CRITICAL
api.company.com Discovered 2 hours ago Auto-detected Database Layer

A SQL injection vulnerability has been detected in the login endpoint. Attackers can bypass authentication and access sensitive database information by injecting malicious SQL code through the username parameter.

9.8
CVSS

Description

SQL injection occurs when user input is incorrectly sanitized, allowing attackers to execute arbitrary SQL commands. This can lead to unauthorized data access, modification, or deletion.

Location

File: /api/auth/login.js
Line: 42
Function: authenticateUser()

Vulnerable Code

const query = `SELECT * FROM users WHERE username = '${username}'`;

Remediation

  • Use parameterized queries or prepared statements
  • Implement input validation and sanitization
  • Use ORM frameworks that handle SQL escaping
  • Apply principle of least privilege for database accounts
